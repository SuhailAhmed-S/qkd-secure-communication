Overview

Quantum Key Distribution (QKD) enables two parties to establish a mathematically secure cryptographic key using quantum mechanics. This implementation provides the BB84 protocol with integrated privacy amplification and real-time eavesdropper detection.

Key Concept: Unlike classical cryptography which relies on computational hardness, QKD security derives from the laws of physics. Any attempt to intercept quantum states fundamentally alters them, making eavesdropping detectable.

BB84 Protocol

The BB84 protocol, proposed by Bennett and Brassard in 1984, is the most famous QKD protocol. It uses two bases (rectilinear and diagonal) to encode qubits.

Protocol Steps

  1. Qubit Preparation: Alice randomly chooses a basis (rectilinear + or diagonal ×) and prepares qubits with polarization states 0 or 1
  2. Quantum Channel: Alice sends qubits to Bob through a quantum channel (potentially monitored by Eve)
  3. Eavesdropping (Eve): Optional: Eve intercepts and measures qubits, potentially introducing errors
  4. Bob's Measurement: Bob randomly chooses bases and measures received qubits
  5. Sifting: Alice and Bob publicly compare bases (not values) and keep only bits measured with matching bases
  6. QBER Calculation: Both parties sample and check a subset of sifted bits to compute Quantum Bit Error Rate
  7. Privacy Amplification: Apply universal hashing to reduce eavesdropper information from O(n) to O(λ⁻¹)
  8. Encryption: Use the amplified key for symmetric encryption (XOR or other cipher)
Alice → [Qubits] → Quantum Channel → [Measurement] → Bob
↓
Eve (Optional Eavesdropper)

Basis Encoding

Rectilinear Basis (+)

|0⟩ = Horizontal (0°)

|1⟩ = Vertical (90°)

Diagonal Basis (×)

|0⟩ = Diagonal (-45°)

|1⟩ = Anti-diagonal (45°)

QBER Analysis

The Quantum Bit Error Rate (QBER) is the key security metric in QKD. It indicates whether eavesdropping is occurring.

QBER Thresholds

QBER = (Number of Errors in Sifted Key) / (Total Sifted Key Bits)

QBER < 11%

SECURE ✓

No significant eavesdropping detected. Key is safe to use.

QBER = 11-25%

SUSPICIOUS ⚠

Indicates possible presence of Eve. Protocol continues cautiously.

QBER > 25%

COMPROMISED ✗

Definite eavesdropping detected. Abort protocol and investigate.

QBER ≈ 25% (with Eve)

EXPECTED WITH EVE

Theoretical prediction when Eve eavesdrops without strategy.

⚠ Security Note: QBER calculation uses a publicly announced subset of bits. These bits must be discarded from the final key, as Eve has seen their values.

Privacy Amplification

Privacy amplification reduces the information Eve has gained about the final key through universal hashing and randomness extraction.

The Problem

Eve may have partial information about the sifted key even after QBER testing. With QBER ≈ 25% (Eve guesses wrong 25% of the time), Eve still knows ~75% of key bits. Privacy amplification extracts a smaller key on which Eve's advantage is negligible.

Universal Hashing (Toeplitz)

Uses Toeplitz matrices for ε-universal hashing with collision probability ≤ 2^(-λ):

T = Random Toeplitz matrix (λ × n)
Amplified_Key = T · Key (modulo 2)

SHA-256 Extraction

Alternative using counter-mode SHA-256 for cryptographic-strength randomness extraction:

For i in range(λ/256):
  block[i] = SHA256(key || counter=i)
Amplified_Key = Concatenate all blocks

Compression Based on QBER

The amplification method adapts based on channel quality:

0-2% QBER → 66% compression (strong channel)
2-5% QBER → 50% compression (good channel)
5-11% QBER → 30% compression (weak channel)
>11% QBER → Abort protocol
✓ Security Guarantee: After amplification, Eve's advantage drops to at most 2^(-λ) where λ is the security parameter (default 128).

Security Analysis

Information-Theoretic Security

BB84 provides unconditional security against an eavesdropper with unlimited computational power. Security does not depend on hardness assumptions.

Quantum No-Cloning Theorem

Eve cannot clone unknown quantum states (no-cloning theorem). Any measurement attempt collapses the state, causing detectable errors in exactly 50% of bases (when she guesses wrong).

Eavesdropper Detection

  • Eve guesses the basis correctly 50% of the time
  • When she guesses wrong, her measurement produces random results
  • This causes bit errors in 50% of her wrong-basis measurements
  • Overall: Eve introduces ~25% QBER if detected
  • Protocol aborts when QBER > 11%, detecting Eve with high confidence

Privacy Amplification Guarantee

Theorem (Vazirani-Widerson): After universal hashing by ε-universal family, Eve's Shannon entropy about the final key increases by log(1/ε). With ε ≤ 2^(-128), Eve gains negligible advantage.

Conclusion: For 100-qubit basis agreement, ~50 sifted bits → ~25 amplified bits with 128-bit security parameter provides practical quantum-secure key exchange.

📖 Overview

Quantum Key Distribution (QKD) represents a revolutionary approach to secure communication by leveraging the fundamental principles of quantum mechanics. This implementation focuses on the BB84 protocol, one of the first and most widely studied quantum cryptographic protocols.

The BB84 protocol enables two communicating parties to securely generate a shared secret key by transmitting quantum bits (qubits) encoded using photon polarization states. The inherent properties of quantum mechanics, such as the no-cloning theorem and measurement disturbance, ensure that any eavesdropping attempt introduces detectable errors in the communication channel.

Key Concepts

  • Quantum Bits (Qubits): Fundamental units of quantum information
  • Basis Reconciliation: Public comparison of measurement bases
  • Sifting: Keeping only bits where bases matched
  • QBER: Quantum Bit Error Rate for security detection
  • No-Cloning Theorem: Impossible to copy unknown quantum states
  • Measurement Disturbance: Measuring changes the quantum state

🔄 BB84 Protocol

The BB84 protocol consists of six main steps:

Step 1: Qubit Preparation

Alice generates random bits and selects random measurement bases for each qubit:

  • Rectilinear Basis (+): 0→|0⟩, 1→|1⟩
  • Diagonal Basis (×): 0→|+⟩, 1→|−⟩

Step 2: Quantum Transmission

Alice sends the encoded qubits through the quantum channel to Bob. Eve may optionally intercept and measure.

Step 3: Random Measurement

Bob randomly selects measurement bases for each received qubit and records the measurement results.

Step 4: Basis Reconciliation

Alice and Bob publicly compare their basis choices (not the bits) over an authenticated classical channel.

Step 5: Sifting

Both parties keep only the bits where their bases matched. This forms the "sifted key."

Step 6: QBER Calculation

Calculate the quantum bit error rate to detect eavesdropping. If QBER exceeds the threshold (11%), abort communication.

🔒 Security Analysis

Information-Theoretic Security

The BB84 protocol provides unconditional security based on the laws of quantum mechanics, not on computational assumptions. This means it remains secure even against quantum computers.

Eavesdropping Detection

When Eve attempts to intercept and measure qubits:

  • Eve uses the wrong basis ~50% of the time
  • This introduces ~12.5% errors in the sifted key
  • Combined with Bob's measurement errors, QBER rises to ~25%
  • This elevation is easily detected and indicates eavesdropping
Scenario QBER Status Action
Clean Channel (No Eve) 0% ✓ Secure Proceed with encryption
With Eve (Intercept-Resend) ~25% ✗ Detected Abort communication
Threshold 11% Decision Point Abort if exceeded

Security Guarantees

✓ Unconditional Security: Proven secure against quantum adversaries
✓ No Computational Assumptions: Security based on quantum mechanics
✓ Eavesdropping Detection: Any unauthorized access is detectable
✓ Perfect Secrecy: One-time pad style encryption

📊 QBER Calculation

Definition

The Quantum Bit Error Rate (QBER) measures the fraction of bits that differ between Alice's and Bob's sifted keys:

QBER = (number of differing bits) / (total sifted bits)

Interpretation

  • QBER ≈ 0%: Clean channel, no eavesdropping
  • QBER ≈ 25%: Eavesdropping detected (Eve present)
  • QBER < 11%: Channel considered secure (threshold)
  • QBER ≥ 11%: Likely eavesdropping, abort communication

Threshold Justification

The 11% threshold accounts for:

  • Quantum noise in real systems (~1-2%)
  • Environmental decoherence (~1-2%)
  • Detector imperfections (~1-2%)
  • Statistical variance (~3-4%)

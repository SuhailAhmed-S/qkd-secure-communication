📚 QKD BB84 Documentation

Welcome to the comprehensive guide for Quantum Key Distribution using the BB84 protocol. This documentation explains the fundamental concepts, mathematical principles, and practical implementation of quantum cryptography.

🔬 What is Quantum Key Distribution?

Quantum Key Distribution (QKD) is a method of secure communication that uses quantum mechanics to guarantee the security of cryptographic keys. Unlike traditional cryptography, QKD provides information-theoretic security - security that is guaranteed by the laws of physics rather than computational complexity.

Key Insight: In quantum mechanics, measuring a quantum system inevitably disturbs it. This fundamental property allows legitimate users to detect eavesdropping attempts.
🎯 The BB84 Protocol

The BB84 protocol, named after its inventors Charles Bennett and Gilles Brassard in 1984, is the first and most widely studied QKD protocol. It uses single photons to transmit cryptographic keys.

Protocol Overview
Alice prepares qubits: Alice randomly chooses bits (0 or 1) and randomly chooses measurement bases (+ or ×) for each qubit.
Alice sends qubits: Alice sends the prepared qubits to Bob through a quantum channel.
Bob measures qubits: Bob randomly chooses measurement bases and measures each received qubit.
Public discussion: Alice and Bob publicly compare their chosen bases (but not the bit values).
Key sifting: They keep only the bits where they used the same basis, discarding the rest.
Error estimation: They sample a portion of their sifted key to estimate the quantum bit error rate (QBER).
Privacy amplification: If QBER is below threshold, they perform privacy amplification to create the final secure key.
⚛ Quantum Mechanics Fundamentals
Photon Polarization States

The BB84 protocol uses the polarization of single photons as quantum bits (qubits). Photons can be polarized in different ways:

↕
Vertical (0°)
Bit: 0, Basis: +
↔
Horizontal (90°)
Bit: 1, Basis: +
⤢
Diagonal (45°)
Bit: 0, Basis: ×
⤡
Anti-diagonal (135°)
Bit: 1, Basis: ×
Measurement Uncertainty

The key quantum property exploited by BB84 is the Heisenberg uncertainty principle for measurement. When Eve tries to measure a qubit in the wrong basis, she has a 50% chance of getting the wrong result, introducing detectable errors.

P(error|Eve intercepts) = 25%

This means that if Eve intercepts and re-sends all qubits, the QBER will be exactly 25%, which is well above the security threshold.

🛡️ Security Analysis
Quantum Bit Error Rate (QBER)

The QBER is the fraction of bits that differ between Alice and Bob after basis reconciliation:

QBER = (Number of errors) / (Total sifted bits)
0%
No Eavesdropping
~5%
Channel Noise
25%
Full Eavesdropping
>11%
Security Threshold
Security Threshold

The BB84 protocol is secure as long as QBER < 11%. Above this threshold, the key is considered compromised and should be discarded.

🔒 Information-Theoretic Security

BB84 security is based on the laws of quantum physics, not computational assumptions. Even with unlimited computing power, an eavesdropper cannot break the encryption without being detected.

Real-World Considerations

In practice, QKD systems face several challenges:

  • Photon loss: Photons can be absorbed or scattered
  • Detector efficiency: Imperfect photon detection
  • Channel noise: Environmental interference
  • Distance limitations: ~100-200km for fiber optics
